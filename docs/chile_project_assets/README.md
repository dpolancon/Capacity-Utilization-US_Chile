# Chile Project Assets

This package reorganizes the uploaded `Chile_Assets` bundle by **methodological function** rather than by origin project. It is designed to be repo-ready and easier to reuse across the WS corridor, AR corridor, and cross-corridor drafting work.

## Folder logic

- `00_index/` — master index and integration log
- `01_source_control/` — ledgers, bibliographic control, corpus backbone maps
- `02_argument_architecture/` — argument maps, periodization, evidence matrices, reframing workbench
- `03_manuscript_reservoir/` — prose-rich draft material; useful but not governance-first
- `04_communication/` — abstracts, talks, handouts
- `05_links_to_corridors/` — notes that route these assets into the WS corridor, AR corridor, and overlap work
- `06_original_bundle_mirror/` — untouched mirror of the original bundle structure for traceability

## Use rule

Treat this package mainly as a **project-architecture bundle**, not as a raw evidentiary source corpus.
The most valuable artifacts are the ledgers, argument/evidence matrix, periodization sheet, source-integration map, and reframing workbench.
Use prose-heavy notes as reservoirs, not as default control layers.
