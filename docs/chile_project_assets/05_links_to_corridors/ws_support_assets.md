# WS Support Assets

This note routes the reorganized Chile project assets into the **WS corridor** workflow.

## How to use this folder for the WS corridor

Treat these assets as a **support and control layer**, not as a substitute for the WS source corpus.
Their main function is to discipline periodization, source-role classification, argument/evidence matching, and overlap handling.

## Assets that directly support the WS corridor

### 1. Source-control assets

- `../01_source_control/ledger_de_fuentes.md`
  - Best use: classify sources by analytic function rather than alphabetically.
  - WS contribution: helps separate structural, historiographic, evidentiary, and contextual materials.

- `../01_source_control/bibliografia_y_auditoria.md`
  - Best use: bibliographic verification and reference cleanup.
  - WS contribution: keeps the corridor traceable and reduces citation drift.

- `../01_source_control/corpus_backbone_maps/source_integration_map_1925_1958.md`
  - Best use: long-run historical backbone and pre-UP sequencing.
  - WS contribution: helps situate early institutional and social transformations without letting them swallow the corridor.

### 2. Argument-architecture assets

- `../02_argument_architecture/argument_evidence_matrix.md`
  - Best use: match claims to required evidence type.
  - WS contribution: prevents the corridor from drifting into narrative without mechanisms.

- `../02_argument_architecture/periodization_control_sheet.md`
  - Best use: standardize the analytical work each historical period must perform.
  - WS contribution: useful for coding external pressure, diplomatic narrowing, and conjunctural shifts by period.

- `../02_argument_architecture/mapa_argumental.md`
  - Best use: reverse-outline logic.
  - WS contribution: shows where WS mechanisms can be inserted without turning into an “external factors” appendix.

- `../02_argument_architecture/dissertation_reframing_workbench.md`
  - Best use: repurpose legacy material into current dissertation architecture.
  - WS contribution: helps adapt older arguments into corridor-compatible framing.

## Recommended WS workflow use

1. Use the WS source corpus and source notes to extract mechanisms and observables.
2. Use the source-control assets here to verify source roles and bibliographic stability.
3. Use the argument-architecture assets here to decide where each mechanism belongs in the broader Chile/UP architecture.
4. Use overlap-sensitive assets to route materials toward WS-only, AR-only, overlap, or hinge status.
5. Keep manuscript and communication artifacts downstream from classification work.
