# WS Corridor Corpus Workspace

This subtree is the working corpus space for the WS corridor of the dissertation. It is organized for intake, traceability, mapping, and later adjudication rather than for prose drafting.

## Folder map
- `00_admin/` administrative records, manifests, logs, and registry files
- `01_sources_raw/flat_current_corpus/` raw source files preserved with original filenames
- `02_notes/sources/` future per-source notes
- `02_notes/dimensions/` ontology stubs for WS dimensions
- `02_notes/mechanisms/` mechanism stubs for cross-source comparison
- `03_bibliography/` resolved and unresolved BibTeX holding files
- `04_mappings/` source, dimension, mechanism, observable, and variable mapping tables
- `05_prompts/` reusable prompt scaffolds for intake workflows
- `06_exports/` later exported tables or synthesis outputs

## Workflow principle
`source -> dimension -> mechanism -> observable -> possible variable / indicator -> evidentiary status`

## Division of labor
- `repo/Codex = intake machine`
- `chat = adjudication machine`
