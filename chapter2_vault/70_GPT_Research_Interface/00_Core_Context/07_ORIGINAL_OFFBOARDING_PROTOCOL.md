# Original Offboarding Protocol

The project rescue must preserve:

- decisions;
- architecture;
- active files and paths;
- theoretical commitments;
- methodological locks;
- unresolved problems;
- next actions;
- reusable prompts;
- produced artifacts;
- mistakes to avoid.

Definition of done:

A future user should know what the project was, what was decided, where the files live, the strongest current formulation, what not to repeat, the next action, and how to restart.
