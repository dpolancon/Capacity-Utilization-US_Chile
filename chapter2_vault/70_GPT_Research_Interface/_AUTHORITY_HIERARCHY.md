# Authority Hierarchy

Use this order whenever sources conflict.

1. Live repository stage outputs and validation files.
2. Active stage specification contracts.
3. Current repository parking manifest and current-state notes.
4. Decision ledgers.
5. Empirical-object, method, and lineage registries.
6. Reviewed offboarding narrative files.
7. Internal drafts and advisor presentations.
8. Chat-derived summaries.
9. AI-generated research notes and exploratory reports.

## Conflict rule

A lower-ranked source may help locate evidence. It may not override a higher-ranked source without explicit review.

## Numerical rule

Any coefficient, sample size, validation count, or commit hash must be checked against the live repository before publication.

## Conceptual rule

Theoretical locks may be reopened only through a documented decision change, not through incidental code behavior.
